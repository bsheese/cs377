- [ ] Statsmodels is incompatible with recent versions of the NumPy library
- [ ] Adding a constant column runs faster in Statsmodels than in other libraries

- [ ] The notebook author preferred this method based on historical computing conventions

### In time-series data, observations close in time tend to have similar values. Which LINE assumption might this violate?

- [ ] Normality — time-series residuals are never normally distributed

## 17_1_4: Influence & Leverage

### Which combination makes a point truly dangerous for regression?
- [x] High leverage AND large residual together

- [ ] Low leverage combined with a very large residual
- [ ] High correlation between the point's x-value and the target variable

- [ ] The model's predictions were incorrect until the dataset was retrained
- [ ] This slope change proves the underlying model was fundamentally broken

### Cook's Distance combines leverage and residual disagreement. Why is this combination necessary?
- [x] Only points with both unusual x-value AND poor prediction are truly problematic

- [ ] Cook's Distance is the only statistically valid influence diagnostic available
- [ ] A high Cook's Distance score automatically means the point should be removed

- [ ] Huber regression eliminates the need for any data cleaning or outlier detection

## 17_1_5: Transformations

### The Gapminder data shows diminishing returns: each dollar of GDP predicts less life expectancy as GDP grows. What causes this curve?
- [x] The derivative of log(x) is 1/x, steep when x is small and flat when x is large
- [ ] Wealthier countries systematically overreport life expectancy in national statistics
- [ ] Linear regression cannot handle any relationship that exhibits curvature
- [ ] The relationship is linear up to a certain GDP threshold, then becomes non-linear

### Why does a log transformation often fix BOTH linearity AND heteroscedasticity violations?
- [x] Log compression affects both the mean structure and the variance structure simultaneously
- [ ] A transformation can only correct one assumption violation at a time
- [ ] Heteroscedasticity does not exist in log-transformed data by construction
- [ ] Satisfying linearity automatically implies that equal variance holds as well

### In the Gapminder model, the coefficient for log(GDP) is 5.8. How do you interpret this slope?
- [x] A 1% increase in GDP is associated with roughly 0.058 additional years of life
- [ ] Doubling GDP is associated with an increase of 5.8 years in life expectancy
- [ ] A one-log-unit increase in GDP adds exactly 5.8 years to life expectancy
- [ ] GDP per capita explains 5.8% of the total variation in life expectancy

### In a log-level model (log(y) = β₀ + β₁*x), what does β₁ = 0.05 mean in plain English?
- [x] A 1-unit increase in x corresponds to ~5% increase in y
- [ ] A 5% increase in y requires a one-unit change in the x variable
- [ ] y doubles each time x increases by 0.05 units in the model
- [ ] x and y have a 5% Pearson correlation in the original data

### What does Box-Cox transformation do, and what does λ = 0 suggest?
- [x] Finds the optimal λ to maximize normality; λ = 0 suggests a log transformation
- [ ] Applies the same power transformation to all features simultaneously
- [ ] Standardizes the target variable to have zero mean and unit variance
- [ ] Automatically identifies and removes outliers from the dataset before fitting

### After log-transforming the target, your model achieves R² = 0.65 (vs. 0.46 before). Is this a real improvement?
- [x] The R² values are not comparable across different y-scales; compare residual patterns instead
- [ ] Yes — a higher R² value on any scale indicates a genuinely better model
- [ ] No — log transformations mechanically inflate R² without improving predictions
- [ ] Only if the improvement in R² exceeds a threshold of 20 percentage points

### After fitting a log-transformed model, predictions are in log-scale. How do you back-transform?
- [x] Exponentiate the prediction; apply Jensen's correction: exp(pred) × exp(σ²/2)
- [ ] Take the exponential of each log-scale prediction without any correction factor
- [ ] Divide each prediction by 2.718 (Euler's number) to reverse the log transform
- [ ] Back-transformation is not possible for models fitted on a log-transformed target

### Why is Jensen's correction important when back-transforming log-level predictions?
- [x] Without it, exp(predicted log value) underestimates the true expected value
- [ ] It ensures that all back-transformed predictions remain strictly positive values
- [ ] It prevents the model from producing negative predictions after back-transformation
- [ ] It is only required for Box-Cox transformations, not for natural log transforms

### How do the three log models (level-log, log-level, log-log) differ in interpretation?
- [x] They describe percentage vs. absolute changes in different combinations of x and y
- [ ] They are mathematically identical despite using different notation conventions
- [ ] Only one of the three log model variants can be used in any given analysis
- [ ] The choice is made at random; interpretation does not depend on the model form

### Why might a transformation mask an underlying problem instead of fixing it?
- [x] A transformation can hide that the true functional form is genuinely non-linear
- [ ] All well-chosen transformations reveal rather than conceal data problems
- [ ] Only incorrectly chosen transformations can mask underlying model issues
- [ ] Masking structural problems is not possible with any standard regression transformation
